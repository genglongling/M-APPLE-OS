    *   `Operation` class: Represents a single operation with its job, step, machine, duration, start time, and end time.
    *   `machine_schedule`: A dictionary to track the scheduled time slots on each machine.  The key is the machine name, and the value is a list of tuples `(start_time, end_time)` representing the intervals when the machine is busy.
    *   `job_completion_times`: A dictionary to store the completion time of each job, ensuring job precedence.

2.  **`create_operations(job_data)` Function:**
    *   Takes the `job_data` dictionary as input.
    *   Iterates through each job and its steps to create `Operation` objects.
    *   Returns a list of `Operation` objects.

3.  **`schedule_operations(operations)` Function:**
    *   Takes a list of `Operation` objects as input.
    *   Initializes `machine_schedule` and `job_completion_times`.
    *   Sorts the operations based on the Shortest Processing Time (SPT) rule.
    *   Iterates through the sorted operations:
        *   Determines the earliest possible `start_time` for the operation, considering both job precedence and machine availability.
        *   Sets the `start_time` and `end_time` of the operation.
        *   Updates `job_completion_times` with the new completion time for the job.
        *   Updates `machine_schedule` to reflect the newly scheduled operation.
    *   Calculates the `makespan` as the maximum completion time among all jobs.
    *   Returns the scheduled `operations`, `makespan`, and `machine_schedule`.
