    machine_schedules = {machine: [] for machine in range(num_machines)}
    job_pointers = [0] * len(jobs)  # Track current operation for each job
    completed_jobs = [False] * len(jobs)
    schedule = []
    current_time = 0

    # Priority queue to store ready operations (machine, duration, job_id)
    ready_queue = []

    # Initialize ready queue with first operation of each job
    for job_id, job in enumerate(jobs):
        machine, duration = job[0]
        heapq.heappush(ready_queue, (duration, job_id))

    while not all(completed_jobs):
        # Find the next available operation
        if not ready_queue:
            current_time += 1
            continue

        duration, job_id = heapq.heappop(ready_queue)
        machine, duration = jobs[job_id][job_pointers[job_id]]

        # Check for machine availability
        start_time = current_time
        machine_schedule = machine_schedules[machine]
        for scheduled_op in machine_schedule:
            if start_time < scheduled_op['End Time']:
                start_time = scheduled_op['End Time']  # Delay until machine is free

        end_time = start_time + duration

        # Update machine schedule
        machine_schedules[machine].append({
            'Job': f'Job{job_id + 1}',
            'Step': job_pointers[job_id] + 1,
            'Machine': f'Machine{machine}',
            'Start Time': start_time,
            'End Time': end_time,
            'Duration': duration
        })
        machine_schedules[machine].sort(key=lambda x: x['Start Time']) #keep sorted

        # Add operation to schedule
        schedule.append({
            'Job': f'Job{job_id + 1}',
            'Step': job_pointers[job_id] + 1,
            'Machine': f'Machine{machine}',
            'Start Time': start_time,
            'End Time': end_time,
            'Duration': duration
        })

        # Advance to the next operation in the job
        job_pointers[job_id] += 1

        # If the job is complete, mark it as such
        if job_pointers[job_id] == len(jobs[job_id]):
            completed_jobs[job_id] = True
        else:
            # Add the next operation to the ready queue
            next_machine, next_duration = jobs[job_id][job_pointers[job_id]]
            heapq.heappush(ready_queue, (next_duration, job_id))

        current_time = max(current_time, end_time)

    makespan = max(op['End Time'] for op in schedule)

    return makespan, schedule

**Explanation of the Code and Algorithm:**

1.  **`solve_jssp(jobs)` Function:**
    *   Takes the `jobs` data as input.
    *   Initializes `machine_schedules`: A dictionary to store the schedule for each machine.  The keys are machine IDs (0-19), and the values are lists of scheduled operations (dictionaries).  This is crucial for preventing machine conflicts.
    *   Initializes `job_pointers`: A list to keep track of the next operation to be scheduled for each job.
    *   Initializes `completed_jobs`: A list to track which jobs have been fully scheduled.
    *   Initializes `schedule`: A list to store the complete schedule of all operations.
    *   Initializes `current_time`: Represents the current time in the schedule.
    *   Initializes `ready_queue`: A priority queue (using `heapq`) to store operations that are ready to be scheduled. The priority is based on the duration of the operation (shortest processing time first).

2.  **Initialization of `ready_queue`:**
    *   The first operation of each job is added to the `ready_queue`.  The `heapq` module ensures that the operation with the shortest duration will be processed first.

3.  **Main Scheduling Loop (`while not all(completed_jobs)`):**
    *   The loop continues until all jobs are completed.
    *   **`heapq.heappop(ready_queue)`:** Retrieves the operation with the shortest duration from the `ready_queue`.
    *   **Machine Availability Check:** This is the core of the conflict resolution.  It iterates through the `machine_schedules` for the machine required by the current operation. If there's an overlap (i.e., `start_time < scheduled_op['End Time']`), the `start_time` is advanced to the end time of the conflicting operation, effectively delaying the current operation.
    *   **Update `machine_schedules`:** The scheduled operation is added to the `machine_schedules` list for the corresponding machine. The list is then sorted by start time to maintain the correct order of operations on the machine.
    *   **Add to `schedule`:** The scheduled operation is added to the overall `schedule` list.
    *   **Advance Job Pointer:** The `job_pointers` list is updated to point to the next operation in the job.
    *   **Add Next Operation to `ready_queue`:** If the job is not complete, the next operation is added to the `ready_queue`.
    *   **Update `current_time`:** The `current_time` is updated to the maximum of its current value and the `end_time` of the scheduled operation. This ensures that the schedule progresses correctly.

4.  **Calculate Makespan:**
    *   After the loop completes, the makespan is calculated as the maximum `End Time` among all operations in the `schedule`.

5.  **Output:**
    *   The `makespan` and the `schedule` are printed in the required format.
