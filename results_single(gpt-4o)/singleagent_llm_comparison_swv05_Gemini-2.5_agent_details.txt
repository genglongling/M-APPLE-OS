1.  **Initialization:**
    *   A list of all operations is created.
    *   A list of available machines is created.
    *   A schedule (initially empty) is created to store the start and end times of each operation.
    *   A `machine_schedule` dictionary is created to track the availability of each machine.  Initially, all machines are available from time 0.

2.  **Scheduling Loop:**
    *   The algorithm iterates until all operations are scheduled.
    *   **Priority Rule (SPT):**  The list of unscheduled operations is sorted based on the shortest processing time (duration).
    *   **Machine Availability Check:** For each operation in the sorted list, the algorithm checks if the required machine is available at the time required for the operation.
    *   **Conflict Resolution (EDD):** If multiple operations are waiting for the same machine, the operation from the job with the earliest due date (EDD) is selected.  The due date is estimated as the sum of the remaining processing times for that job.
    *   **Scheduling:** If the machine is available, the operation is scheduled. The start time is set to the machine's available time, and the end time is calculated by adding the operation's duration to the start time. The `machine_schedule` is updated to reflect the machine's new availability time.
    *   **Job Precedence:** After scheduling an operation, the next operation in the same job is added to the list of unscheduled operations (if it's not already there).
    *   **Makespan Update:** The makespan is updated to be the maximum completion time of all scheduled operations.

3.  **Output:**
    *   The final makespan is printed.
    *   The structured schedule, containing the job, step, machine, start time, end time, and duration for each operation, is printed.

*   **Heuristic Nature:** This solution is based on heuristics and does not guarantee the absolute optimal makespan. Other scheduling algorithms (e.g., genetic algorithms, simulated annealing, tabu search) could potentially find better solutions, but they are more computationally intensive.

*   **Dataset Specific Tuning:** The performance of different heuristics can vary depending on the specific characteristics of the dataset. It might be possible to improve the makespan by experimenting with different priority rules and conflict resolution strategies.
*   **Local Optima:** Heuristic algorithms can get stuck in local optima. Techniques like random restarts or more sophisticated search strategies can help to escape local optima.
