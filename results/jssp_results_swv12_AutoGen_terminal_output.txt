3. Ensure no machine conflicts (one operation per machine at a time)

def spt_schedule():
    ready_operations = get_initial_ready_operations()  # First op of each job
    current_time = 0
    machine_availability = [0] * 10  # When each machine becomes free
    
    while ready_operations:

        ready_operations.sort(key=lambda op: op.processing_time)
        
        # Select shortest processing time operation
        selected_op = ready_operations.pop(0)

        end_time = start_time + selected_op.processing_time

## Key Scheduling Decisions

1. **Priority Rule**: SPT minimizes average flow time and keeps machines busy with quick operations
2. **Tie-breaking**: When processing times are equal, prioritize by:
   - Earliest available start time
   - Job index (lower number first)
   - Operation index within job

3. **Constraint Handling**:
   - Job precedence: Operation i+1 cannot start until operation i completes
   - Machine capacity: Only one operation per machine at any time
   - No preemption: Once started, operations cannot be interrupted

## Expected Benefits of SPT Rule

- **Advantages**:
  - Simple to implement and understand
  - Good for minimizing average completion time
  - Keeps machines busy with shorter operations
  - Provides good baseline for comparison

- **Limitations**:
  - May not minimize makespan optimally
  - Longer operations might be delayed significantly
  - Doesn't consider global optimization
